# Docker Container Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Package the Dreadnought web server as a production-ready Docker image for cloud deployment (Fly.io, Railway, Render).

**Architecture:** Multi-stage Dockerfile — build stage installs deps with uv into `.venv`, runtime stage copies only the venv and app source. Three config values (PORT, LOG_LEVEL, DATABASE_PATH) become env vars with sensible defaults. A `/health` endpoint is added for cloud platform readiness checks.

**Tech Stack:** Python 3.12, FastAPI, uvicorn, uv, aiosqlite, Docker multi-stage build

## Task 5: Create Dockerfile

**Files:**
- Create: `Dockerfile`

- [ ] **Step 1: Create Dockerfile at project root**

```dockerfile
# ── Build stage ───────────────────────────────────────────────────────────────
FROM python:3.12-slim AS builder
WORKDIR /app

COPY --from=ghcr.io/astral-sh/uv:latest /uv /bin/uv

COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev --no-install-project

# ── Runtime stage ─────────────────────────────────────────────────────────────
FROM python:3.12-slim AS runtime
WORKDIR /app

COPY --from=builder /app/.venv /app/.venv
COPY . .

RUN useradd -m appuser \
    && mkdir -p /app/data/db \
    && chown -R appuser /app

USER appuser

ENV PATH="/app/.venv/bin:$PATH"

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=5s --retries=3 \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:${PORT:-8000}/health')"

CMD ["python", "-m", "web.main"]
```

## Deployment Notes

For cloud platforms (Fly.io, Railway, Render):

1. **Build:** Platform builds image from `Dockerfile` automatically on push.
2. **Port:** Set `PORT` env var to match whatever port the platform assigns (or leave at 8000 if the platform doesn't inject `PORT`).
3. **Database volume:** Mount a persistent volume at `/app/data/db` and set `DATABASE_PATH=/app/data/db/dreadnought.db`.
4. **Health check:** Platform will poll `GET /health` every 30s. Container is healthy when it returns 200.
5. **Single instance:** SQLite is single-writer — run exactly one instance per volume.
